How to Use This Tool

This tool helps you convert Google Docs into GitHub issues. Please follow these guidelines:

Ensure your Google Doc includes clear markers:

New Issue: [Issue Title] [Issue description and details] Update Issue: [Comment content]
  • Headings and text formatting
  • Bullet points and numbering
  • Basic text styling (bold, italic)
  • Links and references

  • Currently not supported: tables, graphs, images

About QuickGit & IssueWeaver

QuickGit is a specialized web application that converts Google Docs into GitHub issues. It's designed to streamline the process of creating and managing GitHub issues in bulk from well-formatted Google Documents.

IssueWeaver GitHub Account

The IssueWeaver account (@IssueWeaver on GitHub) is NASA IMPACT's dedicated service account for hosting and managing our GitHub automation applications. This account:

  • Owns and administers our GitHub-based automation tools
  • Maintains consistent access across our organization
  • Acts as the identified creator of automated issues and comments
  • Ensures separation between personal and automated actions
Why This Structure?

Having a dedicated service account (IssueWeaver) for our GitHub applications:

  • Provides clear identification of automated actions
  • Maintains consistent permissions across repositories
  • Allows for future expansion with additional automation tools
  • Keeps automation separate from individual user accounts

Common Issues & Solutions

  • Ensure the document is shared with your account.
  • Check you're signed in to the correct Google account.
  • Ensure that IssueWeaver has write access to your repo.
  • Verify "New Issue:" and "Update Issue:" markers.
  • Check for extra spaces or formatting.
  • Ensure content follows each marker.
